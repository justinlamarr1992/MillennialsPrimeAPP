# iOS 26 SDK Upgrade Plan (Issue #44)

## Context
Apple requires all App Store submissions to be built with the **iOS 26 SDK** by **April 28, 2026**.
For EAS Build, this means using the Xcode 26 build image. The cleanest path is upgrading from
Expo SDK 53 → 54, which ships React Native 0.81 and Xcode 26 support. The minimum iOS deployment
target for users **stays at 15.1** — no user-facing runtime change.

## Current State
- Expo SDK: `~53.0.24` / React Native: `0.79.6`
- iOS deployment target: `15.1` (stays as-is)
- New Architecture: **already enabled** (good — required for Reanimated v4)
- EAS Build: no Xcode version pinned (will default to old image)

## Step-by-Step Plan

### 1. Branch
```
git checkout -b chore/expo-sdk-54-ios26-upgrade
```

### 2. Upgrade Expo SDK and all dependencies
```
npx expo install expo@~54.0.0 --fix
```
This auto-updates:
- `expo`: 53.0.24 → ~54.0.0
- `react-native`: 0.79.6 → ~0.81.x
- All `expo-*` packages to SDK 54 compatible versions
- `react-native-reanimated`: 3.17.4 → 4.x (New Arch already enabled — compatible)

### 3. Update eas.json — pin Xcode 26 production image
File: `eas.json`
```json
"production": {
  "ios": {
    "image": "macos-sequoia-15.6-xcode-26.0"
  }
}
```

### 4. Run expo-doctor to surface any remaining issues
```
npx expo-doctor@latest
```
Fix any flagged incompatibilities before proceeding.

### 5. Regenerate ios/ directory
```
npx expo prebuild --clean
cd ios && pod install
```
The ios/ directory is managed by prebuild — clean regeneration is the correct approach.

### 6. Address known breaking changes

#### expo-file-system (low risk)
- File: `hooks/useProfilePictureUpload.ts`
- Current import: `import * as FileSystem from 'expo-file-system'`
- Uses: `readAsStringAsync`, `EncodingType.Base64`
- If these APIs moved to legacy in SDK 54, update import to `'expo-file-system/legacy'`
- Test mocks in 3 test files would need the same import path update

#### react-native-reanimated v3 → v4 (low risk)
- Files: `components/HelloWave.tsx`, `components/ParallaxScrollView.tsx`
- Uses only basic hooks (`useSharedValue`, `useAnimatedStyle`, `withTiming`, etc.) — all v4 compatible
- Test mock in `__tests__/setup.ts`: update to use v4 mock if `react-native-reanimated/mock` path changes

### 7. Run all tests
```
npm test
```
All tests must pass before building. Fix any regressions from SDK/RN version changes.

### 8. Commit and push
Stage: `package.json`, `package-lock.json`, `eas.json`, `ios/` (full regenerated directory)

### 9. EAS Build + TestFlight
```
eas build --platform ios --profile production --auto-submit
```
Verifies the Xcode 26 image works end-to-end and the build reaches TestFlight.
